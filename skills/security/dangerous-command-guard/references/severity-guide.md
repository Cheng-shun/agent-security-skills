# 四级严重性分类标准

## 级别定义

| 级别 | 标签 | 含义 | 默认决策 |
|------|------|------|---------|
| CRITICAL | 🔴 致命 | 不可逆系统级破坏，数据永久丢失，无法恢复 | 立即拒绝，不提供绕过 |
| HIGH | 🟠 高危 | 可能导致严重安全事故，但可恢复或有替代方案 | 阻止 + 要求手动确认 |
| MEDIUM | 🟡 中危 | 有风险但影响可控，通常有安全替代方案 | 警告 + ok 后继续 |
| LOW | 🟢 低危 | 非最佳实践，但在某些场景下可接受 | 静默审计日志记录 |

## 决策矩阵

取决于两个因素：
1. **可逆性** (Reversibility) — 操作后果是否可以撤销？
2. **影响面** (Blast Radius) — 操作影响的系统/数据范围？

| 影响面 \ 可逆性 | 不可逆 | 可逆 |
|-----------------|--------|------|
| 系统级 (System) | CRITICAL | HIGH |
| 项目级 (Project) | HIGH | MEDIUM |
| 文件级 (File) | MEDIUM | LOW |
| 无影响 (None) | — | PASS |

## 分类示例

### CRITICAL — 不可逆 + 系统级

- `rm -rf /` — 系统立即不可用
- Fork bomb — 系统资源耗尽
- 磁盘格式化 — 数据永久丢失
- 删除 Windows System32 — 系统不可恢复

### HIGH — 可逆 + 系统级 / 不可逆 + 项目级

- `curl|sh` — 可能被攻击者替换 payload
- `git push --force main` — 代码历史丢失（有 reflog 可恢复但麻烦）
- `DROP DATABASE` — 无备份时不可逆
- `chmod 777 /etc` — 安全退化可通过配置管理恢复

### MEDIUM — 可逆 + 项目级 / 不可逆 + 文件级

- `rm -rf .` — 删除当前项目（git 有副本但本地 WIP 丢失）
- 全局包安装 — 可卸载
- 批量文件删除 — 单文件级别影响

### LOW — 可逆 + 文件级

- `git reset --hard` — 丢失未提交更改
- `.env` 文件读取 — 不修改文件

## 升级规则

以下情况可将严重级别**提升一级**：

1. **生产环境** — 任何在生产服务器上的操作 +1 级
2. **无备份** — 无备份确认的数据删除操作 +1 级
3. **未认证来源** — 命令来自未经验证的 URL/用户输入 +1 级
