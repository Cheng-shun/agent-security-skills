# 6 个真实供应链攻击案例

---

## 案例 1：event-stream 事件（2018）

**包名**：`event-stream` (npm, 200万+ 周下载)

**攻击方式**：恶意维护者接管了项目，在 `flatmap-stream` 子依赖中注入代码，窃取比特币钱包私钥。

**被注入代码**：
```javascript
try {
  var http = require('http');
  // 窃取 bitcoin copay 钱包的私钥
  ...
}
```

**依赖审计检测**：
- 阶段 1：发现 `event-stream@3.3.6`
- 阶段 2：无 CVE（攻击通过社会工程学而非技术漏洞）
- ✓ **阶段 3 触发**：子依赖 `flatmap-stream` 下载量异常低、发布时长 < 30 天
- 结果：标记为"可疑传递依赖"

**教训**：不能只审计直接依赖。传递依赖是最隐蔽的攻击面。

## 案例 2：colors.js / faker.js 破坏事件（2022）

**包名**：`colors` (npm, 2000万+ 周下载)

**攻击方式**：维护者因抗议大公司"白嫖"开源，在 `colors@1.4.0` 中故意引入无限循环，导致依赖此包的应用崩溃。

**依赖审计检测**：
- 阶段 2：无 CVE（不是传统漏洞）
- ✓ **阶段 3 触发**：维护者数量 = 1，且版本更新日志可疑
- 结果：标记为"单人维护风险"

**教训**：依赖的"健康度"需要多维度评估，包括维护者数量和发布模式。

## 案例 3：dependency confusion 攻击（2021）

**包名**：多家公司内部包被仿冒

**攻击方式**：攻击者发现某公司内部使用的私有包名（如 `@company/internal-lib`），在公共 npm 注册表中注册同名包并设定更高版本号。CI 构建时自动拉取了恶意版本。

**受影响公司**：Apple、Microsoft、Tesla、Uber 等均受影响。

**依赖审计检测**：
- ✓ **阶段 1 触发**：检测到 `@company/internal-lib` 同时存在于公共和私有源
- ✓ **阶段 3 触发**：包发布 < 30 天，下载量 < 100，与已知内部包名冲突
- 结果：CRITICAL — Dependency Confusion

**教训**：需要明确区分公共和私有依赖的注册表来源。

## 案例 4：ua-parser-js 恶意版本（2021）

**包名**：`ua-parser-js` (npm, 800万+ 周下载)

**攻击方式**：攻击者通过社会工程学获取了发布权限，在 `0.7.29`、`0.8.0`、`1.0.0` 版本中植入挖矿和窃密代码。

**依赖审计检测**：
- 阶段 2：无 CVE（通过 npm 安全公告发现）
- ✓ **阶段 2 补充**：检测到特定版本已被 npm 标记为 deprecated/removed
- 结果：CRITICAL — 版本已被官方移除

**教训**：已下架的版本是最明确的危险信号。

## 案例 5：Python 包 typosquatting（2023）

**包名**：`requets` vs `requests` (PyPI)

**攻击方式**：恶意开发者发布了拼写相似的包名，利用开发者的拼写错误。`requets`（少了一个 's'）包含窃取环境变量的代码。

**依赖审计检测**：
- ✓ **阶段 3 触发**：Levenshtein 距离 = 1，与 `requests` 完全相同功能声称
- 混淆字符检测：无数字替换（纯拼写错误）
- 结果：CRITICAL — Typosquatting

**教训**：即使没有技术漏洞，一个拼写错误就可能让整个项目沦陷。

---

## 案例 6：AGPL 许可证"定时炸弹"（2024）

**包名**：某开源 AI 模型推理库

**攻击方式**：一个热门的 AI 推理库突然从 MIT 切换为 AGPL-3.0。所有依赖此库的商业 SaaS 产品被迫开源全部代码或停止使用。

**依赖审计检测**：
- ✓ **阶段 4 触发**：许可证变更检测（通过 npm registry 的 historical license 数据）
- 当前版本：AGPL-3.0，之前版本：MIT
- 结果：CRITICAL — 商业项目需要立即替换或获取商业许可

**教训**：许可证不是静态的。依赖审计必须是持续的过程。

---

## 统计

| 案例 | 攻击类型 | 如果未被检测的后果 |
|------|---------|------------------|
| event-stream | 维护者接管 | 用户私钥被盗 |
| colors.js | 维护者破坏 | 生产环境崩溃 |
| dependency confusion | 名字空间混淆 | CI/内部代码泄露 |
| ua-parser-js | 账号被盗 | 服务器被控制 |
| requets | 拼写欺诈 | 环境变量泄露 |
| AGPL 切换 | 许可证变更 | 强制开源 / 法律诉讼 |

> 6 个案例中，只有 1 个能通过传统 CVE 扫描发现。其他 5 个都需要**多维度审计**：维护者分析、拼写检测、许可证追踪。
